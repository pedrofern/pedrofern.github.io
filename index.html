Laboratory / Project V (LAPR5) – 20 values

• The LAPR5 project had the objective of developing the architecture and implementation of a pharmacy management.
• The project contemplated components for administering drugs, prescriptions, stock of pharmacies, orders and their monitoring.
• As a basis, we had the Medication and Receipt APIs, performed during the semester in the discipline of ARQSI (Systems Architecture).
• In the end, 4 Back-End were made in Node.js (Express and Sails), 3 Front-End (Angular JS and React), centralized information with Auth0, MongoDB databases, two Linux servers (Monitoring and Parser Prolog) and an API in Node.js for bootstrap.
• The group was formed by 7 people. Each one had several contributions and in the different areas.
• I collaborated, fundamentally, in the Back-end component of pharmacy management, issues related to the adaptability of the GDRP (General Data Protection Regulation) to the whole system and implemented the monitoring system.
• The monitoring system covered 100% of the components. As a complement to this area, implemented daily backups of all databases.
• The project had a duration of 4 weeks in an intensive format and an agile and iterative with four sprints.
• Continuous integration was used in git repositories. Each component had a repository.
• The documentation was in the general system repository wiki on Bitbucket.
• The management of the project was done using the AGILE methodology and through the Trello platform.
• The project included a final presentation and demonstration of the whole system.
